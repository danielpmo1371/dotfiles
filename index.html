Zsh + Bash
Dual-shell support with shared configuration layer
click to expand
Modular design: ~/.zshrc and ~/.bashrc source shared files from config/shell/ for aliases, env, path, and git config. Zsh uses Zap plugin manager with Powerlevel10k prompt. 150+ shared aliases across both shells.
zap powerlevel10k shared aliases
Neovim
LazyVim-based config with curated plugins
click to expand
Built on LazyVim with custom plugin configs for Git integration, AI code completion (NeoCodeium), Lualine statusbar, Mason LSP management, and Gruvbox colorscheme. Custom keymaps for rapid navigation.
lazyvim neocodeium mason gruvbox
Tmux
Session management with TPM plugins
click to expand
Ctrl+E prefix, vi-mode copy, mouse support, status bar on top. Plugins: Gruvbox theme, session resurrect + continuum for persistence, floax for floating panes. 50k line history, zero escape delay.
gruvbox resurrect continuum floax

// installation
Three ways to get started.
bootstrap — one-liner
# Clone and install in one command
curl -fsSL https://raw.githubusercontent.com/danielpmo1371/dotfiles/main/bootstrap.sh | bash
interactive — dialog TUI
# Pick components and packages interactively
./install.sh

# Choose a package profile:
# Minimal → git, curl, wget
# Developer → + nvim, tmux, zsh, node, ripgrep, fzf, fd, bat, jq
# Full → + delta, lsd, zoxide, eza, chafa, htop, btop, gdu...
# Custom → hand-pick each package
CLI — targeted install
# Install individual components
./install.sh --tools # dev tools
./install.sh --zsh # zsh + zap + p10k
./install.sh --tmux # tmux + TPM + plugins
./install.sh --terminals # ghostty config
./install.sh --claude # claude code CLI
./install.sh --all # everything
